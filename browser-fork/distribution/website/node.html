Anonymous Layer — for operators & developers

The protocol, the reference daemons, the CLI tools, and the Node.js library. AGPL-3.0+. One git repository, one npm install. No browser, no GUI, no telemetry.

PRE-AUDIT TESTNET — read the threat model first. The cryptographic core has not been independently audited. Two protocol versions live in this tree (v0.1 stable single-hop, v0.2 multi-hop in progress). v0.2 binaries refuse to start without an --i-understand-this-is-experimental flag. Do not point at-risk users at this network until the audit closes and the open § 11 gaps in docs/RUNNING-A-NODE.md are resolved. Threat model and current limits →

What's in the package

Daemons

anon-node

Relay / forwarder

Reference relay daemon. Generates a persistent Ed25519 identity, listens for inbound link connections, forwards cells. The thing you run on a VPS to add capacity to the network.

anon-node init <config>
anon-node run <config>

anon-service

Hidden-service publisher

v0.2 hidden-service daemon. Generates a service identity, picks intro points, publishes a descriptor, accepts INTRODUCE2 cells, splices the resulting streams to a local TCP destination.

anon-service init · anon-service publish

anon-mkconsensus

Directory authority

The DA side of v0.2. Generates a DA Ed25519 keypair and signs a consensus from a JSON relay-descriptor file. Pair with the DA Caddy + Docker config under deploy/ to run a public DA.

anon-mkconsensus init · anon-mkconsensus build

anon-rendezvous-proxy

Chained-HS plumbing

Local TCP listener that pipes accepted connections into a fixed .anon rendezvous. Lets you chain two hidden services so the inner content host's IP is never observably linked to the outer address.

Client / CLI tools

anon-socks

SOCKS5 frontend

SOCKS5 server that exposes anon-layer circuits to local apps. The anon-layer tunnel factory wires through the v0.2 runtime; the direct factory is for transport debugging only and is not anonymous.

anon-chat

v0.1 chat client

Interactive single-hop chat over the v0.1 overlay between two nodes that already have each other in their seed list. Useful for smoke-testing a freshly bootstrapped pair.

anon-site-server / -client

text/anon protocol

Reference server + CLI client for the text-oriented anon-site protocol. Serve a directory; fetch and render an anon:// URL. The server speaks plain TCP today; once v0.2 rendezvous is wired it rides on a circuit.

anon-browse

Terminal browser

Single-file ANSI terminal browser for the anon-site network. No GUI dependencies. Numbered links, back/forward history, scroll nav, help screen.

Library

Everything under modules/ is plain ESM and is what the daemons above are built from: the wire codec, the cell dispatcher, the circuit builder, the HS publisher state machine, the DA verifier, the cryptographic primitives. The library and the spec are intentionally small so that an independent re-implementation in another language is a realistic project, not a multi-year ordeal.

import { createNode } from 'anonymous-layer/modules/node/node.mjs';
import { createCircuitBuilder } from 'anonymous-layer/modules/v2-runtime/circuit_builder.mjs';

Module surface is pre-1.0 and will move. Pin the git SHA, not ^x.y.z, until v0.2 ships.

Install

From source (recommended)

All daemons, all CLI tools, the library.
git clone https://github.com/anon-gratis/anonymous-layer
cd anonymous-layer
npm install
./bin/anon-node.mjs --help
./bin/anon-service.mjs --help

npm (git-direct)

Pulls anon-node + anon-chat onto your PATH.
npm install -g \
  github:anon-gratis/anonymous-layer

anon-node --help
anon-chat --help

No anonymous-layer tarball on the npm registry yet — publish lands with the v0.2 cut.

Docker (build from source)

For relays + directory authorities under deploy/.
git clone https://github.com/anon-gratis/anonymous-layer
cd anonymous-layer
docker build -t anon-relay:dev deploy/docker/relay
docker build -t anon-da:dev    deploy/docker/da

A standalone anonymous-layer-node-*.tar.xz release artifact is on the roadmap. Until it lands, the source paths above are the authoritative install routes — the same routes the project uses for its own deployments.

Operator quickstarts

Run a relay

Provision a fresh Debian/Ubuntu VPS, then:

git clone https://github.com/anon-gratis/anonymous-layer
cd anonymous-layer
sudo ./deploy/scripts/bootstrap-relay.sh

The script installs Docker, opens 22/80/443 in ufw, builds anon-relay:dev, lays out /opt/anon-relay/, and prints the next steps (fill in .env, drop in your consensus.bin, docker compose up -d).

Run a directory authority

sudo ./deploy/scripts/bootstrap-da.sh

Mirrors the relay path: builds anon-da:dev, lays out /opt/anon-da/, and waits for you to supply relays.json and DNS for $DA_DOMAIN. See deploy/README.md for the full DA lifecycle.

Host a hidden service

# on the content host
anon-service init   ~/.anon-service
anon-service publish ~/.anon-service \
  --local 127.0.0.1:8080

For a long-running deployment, drop the descriptor-regen systemd templates into /etc/systemd/system/ so the descriptor is re-signed and re-uploaded every 12 h. Without that timer the service appears healthy locally but goes unreachable when the 24 h descriptor expires.

Contributing & alternate implementations

Both the spec and the reference code are AGPL-3.0+; the intent is that participating in the network should not require trusting any particular implementation. The wire format and crypto primitives are documented in docs/SPEC.md independently of this codebase, and the v0.2 draft is structured so a second implementation can be written against it. Issues, RFC-style proposals, and independent implementations are all welcome at github.com/anon-gratis/anonymous-layer.
