Anon Layer — Settings

This extension renders anon:// pages by calling a local bridge (the anon-browse-gui Node process from this repo). Point the extension at the URL the bridge prints when it starts.

How to start a bridge

# From the anonymous-layer repo (replace consensus paths for your DA): bin/anon-browse-gui.mjs \ --consensus tests/consensus.bin \ --da-trust tests/da-trust.txt \ --descriptor tests/descriptor.bin \ --listen 127.0.0.1 --port 1081

The bridge prints a URL like http://127.0.0.1:1081/?token=…. Paste that into the field above and click Parse, then Save.

URL bar access (required for typed .anon addresses)

Firefox does not recognise the .anon TLD natively, so typing foo.anon in the URL bar normally goes to the default search engine (a privacy leak — your .anon hostname gets sent to DuckDuckGo / Google / whoever). To intercept that and open the site through this extension instead, grant the extension permission to see all navigations. This is a one-time click; nothing about the data leaves your machine.

Entry points

  • URL bar: type foo.anon directly (once URL bar access is granted above)
  • URL bar: type anon foo.anon/path (omnibox keyword — always works)
  • Toolbar icon → opens a blank renderer tab
  • Right-click an anon:// link → Open with Anon Layer
  • Clicking an anon:// link on any page is intercepted
